缓存组网----视频与大文件流量引导

视频与大文件流量引导是将在线视频网站的视频点播流量和大文件的流量引导到缓存系统，从而降低出口带宽。本公司的xcache缓存系统通过镜像数据采集文件，支持视频点播播放(请参阅本公司xcache系统）。

以下命令分别为配置缓存地址及启用流量引导。

BRI(config-bri-if)#http cache ip ＜xcache ip address＞

BRI(config-bri-if)#http cache enabl

no http cache enable是流量引导的总开关，若要对部分网段关闭流量引导，可以使用映射中的“h”关键字。在映射中加上h，说明该映射对应的流量取消流量引导。

本功能需要启用NAT。

内核态的HTTP包缓存技术、无IO设计、网页加速

内核态的包缓存技术只针对小网页的加速，无IO设计、针对网页的内容做数据包级别的包缓存。

BRI（config-bri-if）#http web xcache //开启web加速、缓存

BRI（config-bri-if）#http web xcache nocache 5 //5是包的大小、本命令是可选命

有些网页的头部带有no-cache标记，用于说明网页不可以缓存，http web xcache nocache xxx是用于忽略这个标记。比如小于5k以内的网页忽略no-cache标记（即小于5k 的网页只要带no-cache标记就不缓存）。如果没有http web xcache nocache xxx 命令，默认是不缓存带no-cache标记。

本项功能需要大的内存，内存大小要求在16G至128G。
