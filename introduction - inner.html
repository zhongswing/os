BridgeOS系统基本操作

"do" 命令

在非普通模式下，命令命令前缀do可以将当前模式暂时改为普通模式。

例如：BRI(config)#do show running-config

将当前的全局配置模式临时改为普通模式并执行show running-config命令。

"no" 命令

关键字“no”， 用于删除一个配置，置于该配置的前面。例如桥接口中要停用PPPoE服务，命令为BRI(config-bri-if)#no pppoe enable 。

修改密码

BRI(config)#password "<"your password">"

说明：该命令将管理密码改为 "<"your password">"

查看配置

BRI#show running-config [global]

说明：该命令查看当前所有的配置，包括全局配置及各个接口配置。可选global将显示限定在全局的配置。

BRI#show running-config interface {gigabitethernet[0/N] | bridge[0/N] | pppoe[0/N] }

说明：该命令查看接口的配置。可选内容[0/N]将显示限定在接口具体的某个接口。

例如：BRI#show running-config interface pppoe 0/0 显示PPPoE接口第0个接口的配置。

BRI#show nat map N

说明：该命令查看全局配置中编号为N的映射。

BRI#show nat group N

说明：该命令查看全局配置中编号为N的分组。

查看运行状态

BRI#show interface {gigabitethernet[0/N] | bridge[0/N] | pppoe[0/N]}

说明：该命令查看接口的运行状态。可选内容[0/N]将显示限定在接口具体的某个接口。

BRI#show interface bridge[0/N] {nat| cache | pppoe}

说明：该命令查看桥接口下NAT、缓存或者PPPoE服务的运行状态。选内容[0/N]将显示限定在桥接口具体的某个接口。

备注：PPPoE服务在PPPoE接口配置，但桥接口中启用与运行，所以PPPoE服务的运行状态应在桥接口中查看。请参阅后续章节“PPPoE服务的启用”。

保存配置

BRI#write

说明：该命令将当前配置保存至系统。如果对系统做了更改但未保存配置，重启设备后更改将丢失。

说明：该命令查看全局配置中编号为N的分组。

进入全局配置模式

BRI#configure terminal

说明：该命令从普通模式进入全局配置模式。

进入接口配置模式

BRI(config)#interface {gigabitethernet 0/N | bridge 0/N | pppoe 0/N}

说明：该命令从全局配置模式进入接口配置模式。该命令除可以在全局配置模式下使用之外，还可以在接口配置模式下使用。备注：关键字与0/N之间的空格可以省略，例如，pppoe 0/0 可以写成pppoe0/0 。

从当前模式中退出

BRI#exit

说明：该命令可以在任何模式下使用，表示从当前模式中退出。 快捷键：CTRL+z

ping工具

BRI#ping "<"IP address or domain name>

telnet工具

BRI#telnet "<"IP address or domain name>

说明：该命令telnet网络上的一台主机。

配置IP地址与子网掩码

BRI(config-if)#ip address "<"ip address> "<"net mask>

说明：该命令给物理接口配置IP地址及子网掩码。

配置默认网关

BRI(config)#ip default-gateway "<"gate way ip address>

说明：该命令给设备配置默认网关。

首次使用

如果这是设备首次使用，建议为设备设置管理密码和IP地址、网关。方法是使用consol线，通过设备机身consol口连接到用户电脑，打开超级终端，设置超级终端的各项参数（写上各项参数及配截图）。设备初次使用无管理密码。

设置管理密码

进入全局配置模式

BRI#configure terminal

设置管理密码

BRI(config)#password "<"your password>

返回到普通模式

BRI(config)#exit

保存配置

BRI#write
